## СМЫСЛ ПРОЕКТА
мы пишем проект по доступу к погоде через https://openweathermap.org/
пользователь должен иметь возможность выбрать геолокацию (можно запросить через браузер),  и даты (по умолчанию сегодня)
даты от 1 января 1979 года до прогноза на 4 дня вперёд
система должна загрузить данные на выбранные данные ... надо кэшировать запросы, сохранить их в БД, (повторно не запрашивать, если данные уже есть)
вывод на экран показывать максимально круто с точки зрения дизайна, использовать все возможные красивые фишки

## Принятые решения
- БД: MySQL на `sky.sinel.me`, база `pogoda`, пользователь `pogoda` (права только на свою базу + shadow для миграций)
- ORM: Prisma
- Источник данных: **Open-Meteo** (бесплатно, без ключа для некоммерческого использования)
  - архив ERA5: `https://archive-api.open-meteo.com/v1/archive` — почасовые данные с 1940 г., задержка ~5 дней
  - прогноз: `https://api.open-meteo.com/v1/forecast` — до 16 дней вперёд
  - геокодинг (поиск города): `https://geocoding-api.open-meteo.com/v1/search`
  - OpenWeatherMap отброшен (исторические данные на free-плане недоступны), `OPENWEATHERMAP_API_KEY` из `.env` можно убрать
